## CSRF and the state Parameter
A CSRF attack happends when an attacker tries to trick the user into completing some login request that the attacker has created, trying to steal user data. The state parameter is a random string that is generated by the client and sent to the authorization server during the OAuth flow, being a unique secret tied to the user's current session.

If an attacker tries to force the victim to use a malicious OAuth callback, the attacker’s state value will not match the victim’s stored session value. Because of this mismatch, the application rejects the request.
